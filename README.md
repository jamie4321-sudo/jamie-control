# 스낵앤가든 운영관리 — 깃허브 이전판 (1단계: 대시보드 · 크루 · 일정/할일)

기존 GAS 웹앱(HTML+google.script.run)을 정적 프론트(깃허브 페이지) + GAS API 백엔드 구조로 나눈 버전입니다.
구글시트는 그대로 데이터 저장소로 사용합니다.

## 왜 이렇게 바꿨나요
`google.script.run`은 GAS가 직접 서빙하는 HTML 안에서만 동작합니다. 프론트를 깃허브로 옮기면 이 호출 자체가 사라지므로,
Apps Script는 **JSON을 반환하는 API 서버**로만 쓰고, 프론트는 `fetch()`로 그 URL을 호출하는 구조로 바꿨습니다.
지금 이 파일들의 `call('fn', ...)` 함수 시그니처는 예전 코드와 최대한 동일하게 맞춰서, 다음 단계(면담/평가/이슈 등) 이식도 수월합니다.

## 폴더 구조
```
snackgarden-web/
├── index.html          대시보드 / 크루 관리 / 일정·할일 페이지
├── css/style.css        미드나잇 그래파이트 테마
├── js/api.js            GAS 연결 + 데모(로컬) 폴백
├── js/app.js             로그인, 네비게이션, 대시보드
├── js/crews.js           크루 CRUD
├── js/schedule.js        주간 일정 + 할일 달력
└── gas/Api.gs            Apps Script에 추가할 API 라우터
```

## 배포 순서

### 1. Apps Script 쪽
1. 기존 프로젝트(Code.gs, Code_NewModules.gs가 있는 스프레드시트 연결 스크립트)를 엽니다.
2. **기존 `Code.gs`의 `doGet(e)` 함수 이름을 `doGetLegacy(e)` 등으로 바꿔서 비활성화**하세요.
   (같은 이름의 함수가 여러 파일에 있으면 배포 시 마지막 정의가 이깁니다 — `gas/Api.gs`의 `doGet`을 살립니다)
3. 이 저장소의 `gas/Api.gs` 내용을 새 스크립트 파일로 추가합니다.
4. **배포 → 새 배포**
   - 유형: 웹 앱
   - 실행 계정: 나
   - 액세스 권한: **전체 공개(익명 포함)** ← 이게 핵심입니다
5. 배포 후 나오는 `.../exec` URL을 복사해두세요.

### 2. 프론트(깃허브) 쪽
1. `snackgarden-web` 폴더 전체를 새 깃허브 저장소에 올립니다.
2. `js/api.js` 맨 위 `const GAS_URL = '';` 에 1번에서 복사한 `.../exec` URL을 붙여넣습니다.
3. 저장소 설정 → Pages → 배포 브랜치를 선택해 GitHub Pages를 켭니다.
4. 몇 분 후 `https://아이디.github.io/저장소명/` 으로 접속하면 끝입니다.

### 3. 로그인 정보
사용자권한 시트에 등록된 이메일/비밀번호를 그대로 사용합니다. (기존과 동일)

## 데모 모드
`GAS_URL`을 비워두면 자동으로 브라우저 localStorage 기반 데모 모드로 동작합니다.
깃허브에 올리기 전, 로컬에서 디자인/동작을 먼저 확인할 때 유용합니다. (아무 이메일/비밀번호로 로그인 가능)
